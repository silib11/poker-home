# iPhone専用・無料ポーカー（WebRTC）

## 概要
**iPhoneだけ** で動作する **お金をかけない（仮想チップのみ）ポーカー** Web アプリ

- App Store 非公開
- サーバ維持費 0 円
- マンションWi-Fi（端末分離あり）対応
- Safari で URL を開くだけ
- WebRTC による P2P 通信

---

## 技術スタック

### フロントエンド
- HTML5 / CSS / JavaScript（ES6）
- バニラJS（フレームワーク不使用）

### 実行環境
- iOS Safari
- ホーム画面追加で疑似アプリ化

### 通信
- WebRTC DataChannel（P2P）
- Firebase Realtime Database（シグナリングのみ）

### ホスティング
- Vercel（HTTPS / 無料）
- Serverless Functions（TURN認証情報管理）

### セキュリティ
- TURN認証情報は環境変数で管理
- クライアントに直接埋め込まない
- Serverless Functionで動的に取得

- ホストがルーム作成（6桁ID生成）
- 参加者がルームID入力で参加
- WebRTC DataChannel による P2P 通信
- Firebase は接続確立時のみ使用

- スモールブラインド / ビッグブラインド設定
- ホストによるブラインド変更機能

- 2〜7人対応
- ディーラーボタン / SB / BB 自動設定
- フェーズ進行：PREFLOP → FLOP → TURN → RIVER → SHOWDOWN
- プレイヤーアクション：Fold / Check / Call / Raise / All-in

### ベット管理
- 現在ベット額の追跡
- レイズ検出とアクション管理
- オールイン判定
- ベットラウンド完了判定

### サイドポット
- 複数プレイヤーのオールイン対応
- メインポット / サイドポット自動計算
- ポット別の勝者決定

### 役判定（完全実装）
- ロイヤルフラッシュ
- ストレートフラッシュ
- フォーカード
- フルハウス
- フラッシュ
- ストレート
- スリーカード
- ツーペア
- ワンペア
- ハイカード

### UI機能
- プレイヤー情報表示（名前 / チップ / ベット額）
- ポジション表示（BTN / SB / BB）
- コミュニティカード表示

- ポット表示
- 勝者表示とハンド名表示

## ゲームロジック詳細

### PokerGame クラス（core/poker.js）

#### 初期化
- プレイヤー配列管理
- ブラインド設定（SB / BB）
- ディーラーボタン位置

#### 山札管理
- 52枚デッキ生成（♠♥♦♣ × 2〜A）
- Fisher-Yates シャッフル

#### ゲーム開始
- ブラインド徴収（SB / BB）
- 各プレイヤーに2枚配布
- ターン順設定（BB+1から開始）

#### アクション処理
- bet(playerIndex, amount)：ベット / レイズ
- call(playerIndex)：コール
- check(playerIndex)：チェック
- fold(playerIndex)：フォールド

#### ターン管理
- nextTurn()：次のプレイヤーへ
- ベットラウンド完了判定
- 全員acted確認
- オールイン時の自動進行

#### フェーズ進行
- nextPhase()：次フェーズへ
- コミュニティカード配布（3枚 → 1枚 → 1枚）
- ベット額リセット
- ターン順リセット

#### 勝敗判定
- determineWinner()：勝者決定
- calculateSidePots()：サイドポット計算
- evaluateHand(hand)：役評価（数値化）
- getHandName(hand)：役名取得
- チョップ対応（同順位の分配）

## WebRTC 通信設計

### WebRTCManager クラス（src/network/webrtc.js）

#### ホスト側
- createRoom()：ルーム作成
- connectToPlayer(roomId, playerId)：参加者と接続
- broadcast(data)：全員に送信

#### 参加者側
- joinRoom(roomId)：ルーム参加
- send(data)：ホストに送信

#### メッセージ種別
- `join`：参加通知
- `state`：ゲーム状態同期
- `action`：プレイヤーアクション
- `ready`：次ハンド準備完了
- `blinds`：ブラインド更新
