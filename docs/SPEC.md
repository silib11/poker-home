# ポーカーゲーム 完全仕様書

## 概要
iPhone専用・無料テキサスホールデムポーカー（WebRTC P2P通信）

## 技術スタック
- **フロントエンド**: HTML5 / CSS / Vanilla JavaScript (ES6)
- **通信**: WebRTC DataChannel (P2P)
- **シグナリング**: Firebase Realtime Database
- **ホスティング**: GitHub Pages

## アーキテクチャ

### ファイル構成
```
index.html                  # エントリHTML
src/style.css               # スタイル
src/main.js                 # 初期化と配線
src/network/webrtc.js       # WebRTC接続管理
core/poker.js               # ゲームロジック
tests/*.test.js             # テスト
```

### 通信フロー
1. ホストがFirebaseでルーム作成（6桁ID生成）
2. 参加者がルームIDで接続
3. WebRTC接続確立後はP2P通信のみ（Firebaseは使用しない）
4. ホストがゲーム状態を管理・ブロードキャスト

## ゲームルール

### テキサスホールデム
- **プレイヤー数**: 2〜7人
- **初期設定**: バイイン（初期チップ）、SB、BB
- **ディーラーボタン**: 毎ハンド時計回りに移動
- **ブラインド**: SB（ディーラー+1）、BB（ディーラー+2）

### フェーズ進行
1. **PREFLOP**: 各プレイヤーに2枚配布、BB+1から開始
2. **FLOP**: コミュニティカード3枚公開、ディーラー+1から開始
3. **TURN**: コミュニティカード1枚追加
4. **RIVER**: コミュニティカード1枚追加
5. **SHOWDOWN**: 役判定・勝者決定
6. **WINNER**: フォールドで勝利時（手札非公開）

### アクション
- **Fold**: 降りる
- **Check**: ベット0でパス（currentBet = 自分のbet時のみ）
- **Call**: 現在のベット額に合わせる
- **Raise**: ベット額を上げる（最低2倍）
- **All-in**: 全チップをベット

### ベットルール
- **ミニマムレイズ**: currentBet × 2
- **ベット単位**: BB単位
- **オールイン**: チップ不足時は自動的にオールイン扱い

### 役判定（強い順）
1. ロイヤルフラッシュ
2. ストレートフラッシュ
3. フォーカード
4. フルハウス
5. フラッシュ
6. ストレート
7. スリーカード
8. ツーペア
9. ワンペア
10. ハイカード

### サイドポット
- 複数プレイヤーのオールイン時に自動計算
- メインポット + サイドポット1, 2, ...
- 各ポットごとに勝者決定
- チョップ（同順位）時は均等分配

### ゲーム終了条件
- チップ0のプレイヤーは次ハンドから除外
- 1人残った時点でゲーム終了
- ホストが「新しいゲームを開始」で全員復活

## データ構造

### GameState
```javascript
{
    players: [
        {
            id: string,
            name: string,
            chips: number,
            hand: [{suit, rank}],
            bet: number,
            folded: boolean,
            acted: boolean,
            lastAction: string
        }
    ],
    buyin: number,
    sb: number,
    bb: number,
    pot: number,
    currentBet: number,
    dealerIndex: number,
    turnIndex: number,
    phase: 'PREFLOP' | 'FLOP' | 'TURN' | 'RIVER' | 'SHOWDOWN' | 'WINNER',
    community: [{suit, rank}],
    deck: [{suit, rank}]
}
```

### WebRTCメッセージ
```javascript
// 参加
{ type: 'join', name: string }

// プレイヤーID通知
{ type: 'player_id', playerId: string, name: string }

// 状態同期
{ type: 'state', state: GameState }

// ゲーム開始
{ type: 'game_start', state: GameState }

// ゲーム更新
{ type: 'game_update', state: GameState }

// アクション
{ type: 'action', playerId: string, action: string, amount?: number }

// 次ハンド準備
{ type: 'ready_next_hand', playerId: string }

// ブラインド更新
{ type: 'blinds', sb: number, bb: number }

// ゲーム終了
{ type: 'game_over' }

// ゲーム再開
{ type: 'game_restart', state: GameState, buyin: number, allPlayers: [] }
```

## ゲームフロー

### 1. 接続フェーズ
1. ホストがルーム作成（名前、バイイン、SB、BB設定）
2. 参加者がルームID入力で参加
3. ホストが「ゲーム開始」

### 2. ハンド開始
1. ディーラーボタン設定
2. SB/BB徴収
3. 各プレイヤーに2枚配布
4. BB+1からアクション開始

### 3. ベットラウンド
1. 現在のプレイヤーがアクション選択
2. ホストがアクション処理
3. 全員がactedかつベット額一致で次フェーズ
4. 1人以外全員フォールドで即座に勝者決定（WINNER）

### 4. フェーズ進行
1. FLOP: 3枚公開
2. TURN: 1枚追加
3. RIVER: 1枚追加
4. SHOWDOWN: 役判定

### 5. 勝者決定
1. サイドポット計算
2. 各ポットごとに役判定
3. チップ分配
4. 全員に「次のハンドへ」ボタン表示

### 6. 次ハンド準備
1. 全員が「次のハンドへ」クリック
2. チップ0のプレイヤー除外
3. ディーラーボタン移動
4. 新しいハンド開始

### 7. ゲーム終了
1. 1人残った時点で終了
2. ホストが「新しいゲームを開始」で全員復活

## 実装の注意点

### WebRTC
- ホストのみがゲーム状態を管理
- 参加者はアクションをホストに送信
- ホストが処理後、全員にブロードキャスト
- 切断時の再接続機能なし（仕様外）

### ゲームロジック
- core/poker.jsは純粋なロジック（UI非依存）
- src/main.jsがcore/poker.jsとWebRTCを配線
- 状態はホストが一元管理

### UI更新
- renderGame()で毎回全体を再描画
- イベントリスナーは再描画後に再設定
- テーブル背景は固定（再描画不要）

### パフォーマンス
- 対戦相手配置はwindow.innerWidthで動的計算
- スクロール完全無効化
- position: fixed で画面固定

### セキュリティ
- 手札は各プレイヤーのみに送信
- 対戦相手の手札は裏向き表示
- SHOWDOWNで全員の手札公開
